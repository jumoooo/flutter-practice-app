## 문제 3: 앱 실행하기 🚀

### 목표
에뮬레이터 또는 실제 기기에서 앱을 실행하고 핫 리로드 테스트

### 작업 내용
1. 에뮬레이터 실행 또는 실제 기기 연결
2. `flutter run` 명령어로 앱 실행
3. 앱이 정상적으로 실행되는지 확인
4. 코드를 약간 수정하고 핫 리로드 테스트

### 실습: 코드 수정해보기
`lib/main.dart` 파일에서 다음을 수정해보세요:

```dart
// 원래 코드
title: 'Chapter 1 - Flutter 학습',

// 수정 후
title: '나의 첫 Flutter 앱',
```

그리고 터미널에서 `r` 키를 눌러 핫 리로드가 되는지 확인하세요!

### 확인 사항
- [x] 앱이 정상적으로 실행됨
- [x] 핫 리로드가 작동함 (코드 수정 후 즉시 반영)
- [x] 에러 없이 실행됨

## 문제 4: Git 초기 설정 📝

### 목표
Git 저장소를 초기화하고 첫 커밋 만들기

### 작업 내용
1. 프로젝트 루트에서 `git init` 실행
2. `.gitignore` 파일이 있는지 확인 (없다면 생성)
3. 모든 파일을 스테이징하고 첫 커밋 생성

### 명령어 예시
```bash
# Git 초기화
git init

# .gitignore 확인 (이미 있을 수 있음)
cat .gitignore

# 파일 추가
git add .

# 첫 커밋
git commit -m "Initial commit: Flutter 학습 프로젝트 시작"
```

### 확인 사항
- [x] Git 저장소 초기화 완료
- [x] `.gitignore` 파일 확인
- [x] 첫 커밋 생성 완료

## 문제 5: 패키지 추가 연습 📦

### 목표
새로운 패키지를 추가하고 사용하는 방법 익히기

### 작업 내용
1. `pubspec.yaml`에 간단한 패키지 추가 (예: `http` 패키지)
2. `flutter pub get` 실행
3. `lib/main.dart`에서 import 문 추가 (실제 사용은 나중에)

### 예시: http 패키지 추가

#### 1. pubspec.yaml 수정
```yaml
dependencies:
  flutter:
    sdk: flutter
  cupertino_icons: ^1.0.2
  http: ^1.1.0  # 추가
```

#### 2. 패키지 설치
```bash
flutter pub get
```

#### 3. import 문 추가 (main.dart 상단)
```dart
import 'package:flutter/material.dart';
import 'package:http/http.dart' as http;  // 추가
```

### 확인 사항
- [x] `pubspec.yaml`에 패키지 추가
- [x] `flutter pub get` 실행 성공
- [x] import 문 추가 완료
