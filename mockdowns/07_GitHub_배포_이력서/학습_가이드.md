# 🚀 7단계: GitHub 배포 및 이력서 작성

## 🎯 학습 목표
- GitHub 저장소 생성하기
- README 작성하기
- 앱스토어/플레이스토어 배포하기
- 이력서에 추가할 내용 정리하기

---

## 📚 핵심 개념

### 1. GitHub 저장소 생성

1. GitHub에 로그인
2. "New repository" 클릭
3. 저장소 이름 입력
4. "Create repository" 클릭

---

### 2. 로컬 프로젝트를 GitHub에 푸시

```bash
# Git 초기화 (이미 했다면 생략)
git init

# .gitignore 확인
# (이미 있을 것입니다)

# 파일 추가
git add .

# 첫 커밋
git commit -m "Initial commit: Flutter 포트폴리오 앱"

# 원격 저장소 추가
git remote add origin https://github.com/사용자명/저장소명.git

# 브랜치 이름 변경 (필요시)
git branch -M main

# 푸시
git push -u origin main
```

---

### 3. README 작성 가이드

```markdown
# 앱 이름

## 📱 소개
앱에 대한 간단한 소개

## ✨ 주요 기능
- 기능 1
- 기능 2
- 기능 3

## 🛠️ 기술 스택
- Flutter
- Dart
- Provider (상태 관리)
- HTTP (API 연동)

## 📸 스크린샷
![스크린샷](이미지_URL)

## 🚀 실행 방법
```bash
flutter pub get
flutter run
```

## 📦 빌드
```bash
flutter build apk --release
```

## 📄 라이선스
MIT License
```

---

### 4. 앱스토어 배포 (Android)

1. **Google Play Console 가입**
   - https://play.google.com/console

2. **앱 생성**
   - 앱 이름, 설명, 스크린샷 등 입력

3. **APK/AAB 업로드**
   - 빌드한 파일 업로드

4. **심사 제출**
   - 심사 통과 후 배포

---

### 5. 앱스토어 배포 (iOS)

1. **Apple Developer Program 가입**
   - 연간 $99

2. **Xcode에서 빌드**
   ```bash
   flutter build ios --release
   ```

3. **App Store Connect에 업로드**
   - Xcode 또는 Transporter 사용

4. **심사 제출**

---
