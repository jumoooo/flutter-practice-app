## 문제 2: 파라미터 전달하기 📦

### 목표
화면 간 데이터 전달하기

### 작업 내용
1. **리스트 화면 만들기**
   - 여러 항목을 가진 리스트 표시

2. **상세 화면 만들기**
   - 선택한 항목의 정보 표시

3. **데이터 전달**
   - 리스트 항목 클릭 시 해당 데이터를 상세 화면으로 전달

### 힌트
```dart
// 리스트 항목 클릭
ListTile(
  title: Text(item.title),
  onTap: () {
    Navigator.push(
      context,
      MaterialPageRoute(
        builder: (context) => DetailScreen(item: item),
      ),
    );
  },
)
```

### 확인 사항
- [x] 리스트 화면 완성
- [x] 상세 화면에서 전달받은 데이터 표시

## 문제 3: 결과 받기 🔄

### 목표
다음 화면에서 결과를 받아오기

### 작업 내용
1. **편집 화면 만들기**
   - TextField로 텍스트 입력
   - 저장 버튼으로 결과 반환

2. **메인 화면에서 결과 받기**
   - 편집 화면에서 반환한 값 표시

### 힌트
```dart
// 결과 기다리기
final result = await Navigator.push(
  context,
  MaterialPageRoute(builder: (context) => EditScreen()),
);

if (result != null) {
  setState(() {
    savedText = result;
  });
}
```

## 문제 6: Named Routes 사용하기 🗺️

### 목표
이름 있는 라우트로 네비게이션 구현하기

### 작업 내용
1. **라우트 설정**
   - MaterialApp의 routes 속성 사용
   - 3개 이상의 라우트 정의

2. **Named Routes로 이동**
   - Navigator.pushNamed() 사용

### 힌트
```dart
MaterialApp(
  initialRoute: '/',
  routes: {
    '/': (context) => HomeScreen(),
    '/detail': (context) => DetailScreen(),
    '/settings': (context) => SettingsScreen(),
  },
)

// 사용
Navigator.pushNamed(context, '/detail');
```

### 확인 사항
- [x] 라우트 설정 완료
- [x] Named Routes로 이동 작동 확인
